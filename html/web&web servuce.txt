<웹서비스>
구글, 트위터, 인스타 등
하나의 큰 뿌리를 공유한다. 
개념: 우리는 웹서비스로부터 원하는 정보를 얻는다.
웹서비스에게 우리가 필요한 정보를 요청하면, 웹 서비스는 거기에 따른 정보를 준다. 
웹서비스는 우리의 요청에 응답하는 '서버'가 된다. 
클라이언트-서버 관계 : 요청하는 사람과 응답해주는 서버
클라이언트가 서버에게 request 한다.
서버는 클라이언트에게 response한다. 

request의 종류 -> 갖다 줘 = GET / 처리해 줘 = POST
갖다 줘 => 이 정보를 나에게 갖다 줘.
처리해 줘 => 이 정보를 처리해 줘.
무엇을??! HTML 파일이 오고 간다.

P2P = Peer - To - Peer
모두가 서버이자 모두가 클라이언트 관계다.

웹이란?
www
하나의 엄청난 큰 정보의 망.

웹 브라우저란?
웹 서버에서 쌍방향 통신하는 html 문서나 파일과 연동하고 출력하는 응용 소프트웨어이다.
서버와 통신할 때 html이 오가는데 이 html문서를 받아주고 보여주는 역할.
다르 서버와 연결해주는 통로의 역할.\
